The login form is vulnerable to SQL injection, and it is possible to bypass the login by using ' OR 1=1-- as a username.

Before we can dump all the passwords, we need to identify places on the page where data from the login query is used. After logging in, the name of the currently logged-on user is displayed in the top right corner, and it might be possible to dump the data there or in the session cookie. A UNION based injection will be used to dump the passwords.

There are two key requirements that must be met for a UNION based injection to work:

  • The number of columns in the injected query must be the same as in the original query
  • The data types for each column must match the corresponding type
